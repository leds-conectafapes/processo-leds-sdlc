# Discovery

Responsável por entender o problema a ser resolvido, buscando entender o dominio do problema e verificando na literatura se existe documentos que descrevam o problema do cliente e soluções existentes. O Resultado dessa fase é o documento de Discovery e o complemento do SOW, contendo o entendimento do problema, a revisão do SOW e as soluções existentes.

O **Product Manager (PM)** é o responsável por conduzir esta fase. Ele tem como objetivo responder as seguintes questões:

* **Qual o domínio do problema e em qual categoria de problemas ele se encaixa?** Entender o dominio do problema tem como prinicpal objetivo saber conceitos, jargões, dominios e informações relevantes sobre o problema. Por exemplo, se o projeto é sobre gestão de inventário, o domínio do problema é gestão de inventário e a categoria de problemas é gestão de ativos de hardware.

* **Quais os beneficios que se esperam ao resolver esse problema?** ou melhor, para quem já resolveu esse problema, quais foram os beneficios obtidos? Por exemplo, se o projeto é sobre gestão de inventário, os beneficios esperados são a redução de custos com reposição de materiais, maior agilidade na organização e transparência no uso dos recursos.   

> [!TIP]
> Entender os beneficios, podem ser usados como indicadores de sucesso do projeto.
